Level 1 · AI Basics
What AI is, what it can do today, and which tools to consider — all in plain English, no jargon.
What is AI?
🧠
What AI actually is
Plain English

AI is software that understands language — it reads your question, understands what you mean, and gives you a useful answer. It has been trained on billions of documents, so it knows a lot.

Think of it as a super-knowledgeable colleague who reads everything, forgets nothing, and is available 24/7.

⚡
What it can do today
No integration needed

Draft emails, summarise long documents, translate languages, compare options, explain complex policies, write reports, review contracts for red flags, answer policy questions instantly.

You can start using this today — paste text, ask a question, get an answer.

5 agents to build — click each for full details including integrations, day-to-day usage, and build estimates.
💬
PR Intake Agent
Covers steps 1 & 2
Converts natural language requests into structured PRs in Coupa. Validates budget and policy, then immediately cross-checks against active contracts and open POs to avoid unnecessary sourcing.
Coupa APINLP parsingContract DBBudget check

Day-to-day: Requestor types "200 welding rods for Cikarang by April 30." Agent creates PR in Coupa, checks if KTR-2024-0088 already covers this (it does), and flags it as SOCO-matched — no sourcing event needed. Saves ~2 hours of manual work per PR.

Complexity: Medium  ·  Est. build: 3–4 weeks

🔍
Sourcing Event Agent
Covers steps 3, 4 & 5
Bundles PRs into sourcing events, discovers qualified vendors, creates RFQ in Coupa, sends invitations, and auto-chases non-responders on a schedule.
Coupa RFQ APIVendor DBEmail automationScheduler

Day-to-day: Every Monday morning, agent scans all pending PRs, groups bundleable ones, creates sourcing events in Coupa, shortlists vendors, and dispatches RFQs — all before the buyer arrives at the office.

Complexity: Medium-High  ·  Est. build: 4–5 weeks

⚖️
Bid Analysis Agent
Covers steps 6 & 7
Reads all submitted bids, scores on a weighted matrix, generates award recommendation with justification, and prepares negotiation brief for the buyer.
Bid parsingScoring engineHistorical dataReport gen

Day-to-day: When bid deadline closes, agent reads all submissions, scores each on price/delivery/compliance/history, flags anomalies (suspiciously low bids, missing docs), and delivers a ready-to-present award recommendation within minutes.

Complexity: Medium  ·  Est. build: 3–4 weeks

🔄
PO Flip Agent
Covers step 9 — highest priority
Upon approval in Coupa, automatically triggers PO creation in S/4HANA, maps all master data fields, routes through POM approval, and sends confirmation to vendor.
Coupa webhookS/4 BAPIPOM workflowVendor email

Day-to-day: Buyer clicks "Approve" in Coupa. Within 3 minutes: PO is in S/4, POM workflow triggered, vendor receives PO confirmation. Zero manual re-entry. Currently this step takes 1–2 days manually — highest single ROI automation.

Complexity: High (S/4 integration)  ·  Est. build: 5–6 weeks

🚚
Expediting Agent
Covers step 10
Continuously monitors open POs for delivery risk. Reads vendor emails for delay signals, updates S/4 delivery dates, escalates to POA, and drafts chaser emails automatically.
S/4 monitorEmail readerAlert engineAuto-draft

Day-to-day: Every morning, agent scans all open POs with delivery due within 14 days. Cross-checks confirmed dates vs actual vendor responses. Any PO at risk gets a chaser email drafted and POA notified — before anyone has to ask.

Complexity: Medium  ·  Est. build: 3–4 weeks

Recommended build sequence
Month 1 · PO Flip Agent → Month 2 · PR Intake Agent → Month 3 · Sourcing Agent → Month 4 · Bid Analysis + Expediting

Start with PO Flip — highest pain point, fastest ROI, no user behaviour change needed. PR Intake changes how users raise requests, so needs more change management. Sourcing and Bid agents add sophistication last.
